# Персональные программы дронов

**Статус:** done

## Зачем

Раньше пользователь сначала создавал программу в библиотеке, а потом назначал её дрону — лишний шаг. Фича даёт каждому дрону собственное «пространство» для команд с момента создания. Библиотечные программы остаются для переиспользуемой логики.

## Поведение

- У каждого дрона есть **персональная программа** (id = droneEntityId, personal: true, скрыта из библиотеки)
- Дрон может **дополнительно** иметь **назначенную** библиотечную программу
- Если назначена — дрон выполняет её; иначе — персональную
- В редакторе (вкладка DRONE):
  - Без назначения: блок персональной с зелёной обводкой, редактируемый; кнопка «Назначить из библиотеки →»
  - С назначением: синий блок assigned (свёрнут, ↗ → библиотека) + серый блок personal (разворачивается, радио → unassign)
- В библиотеке: у каждой программы чипы назначенных дронов (клик → переход на дрона)

## Критерии готовности

- [x] `personal?: boolean` в `ProgramDef`
- [x] `personalProgramId` и `assignedProgramId` в `ProgramComponent` и `DroneState`
- [x] Миссии 1–4 создают personal-программы для дронов
- [x] `assignProgram` сохраняет `assignedProgramId`, `unassignProgram` очищает
- [x] `programs`-селектор фильтрует personal-программы из библиотеки
- [x] UI вкладки DRONE с двумя блоками и радио-переключением
- [x] UI вкладки LIBRARY с чипами назначенных дронов

## Технические заметки

Модель A+: два взаимоисключающих режима, переключаемых через радио в редакторе.
`personalProgramId` инициализируется пустой строкой в `createDrone.ts`, устанавливается реальным значением в `buildScene()` каждой миссии.
