# Миграция миссий под атомарную семантику

**Статус:** planned

## Зачем

После введения атомарных действий (`docs/features/done/atomic-actions.md`) встроенные миссии 1–4 перестали работать корректно. Их программы и тексты-подсказки построены вокруг старой семантики («MINE добывает всё», «DROP выгружает всё», «CHARGE заряжает до полного»). В новой семантике каждая команда выполняет одну единицу работы: прежние программы либо не делают ничего полезного (дрон движется к шахте один шаг, не добывает, разворачивается), либо используют устаревший формат IF-условий.

## Поведение

Общий паттерн для всех программ — `LOOP` с несколькими `IF`-блоками. Каждый тик цикла дрон проверяет все условия по порядку; несколько условий могут сработать за один тик (например, у шахты: сначала `MINE`, затем первый шаг `MOVE_TO(base)`).

### Базовый паттерн (без зарядки)

```
LOOP {
  IF distance(base) = 0  AND inventory > 0      → DROP
  IF inventory > 0       AND distance(base) > 0  → MOVE_TO(base)
  IF inventory = 0       AND distance(mine) = 0  → MINE
  IF inventory = 0       AND distance(mine) > 0  → MOVE_TO(mine)
}
```

### Расширенный паттерн (с зарядкой)

```
LOOP {
  IF distance(charger) = 0 AND energy < energyMax                      → CHARGE
  IF energy < 30           AND distance(charger) > 0 AND inventory = 0 → MOVE_TO(charger)
  IF distance(base) = 0    AND inventory > 0                           → DROP
  IF inventory > 0         AND distance(base) > 0                      → MOVE_TO(base)
  IF inventory = 0         AND energy >= 30 AND distance(mine) = 0     → MINE
  IF inventory = 0         AND energy >= 30 AND distance(mine) > 0     → MOVE_TO(mine)
}
```

Зарядка — первый приоритет, идёт только при пустом трюме. Пока энергия ниже порога (30), дрон не добывает.

## Критерии готовности

- M1: игрок может собрать программу из 4 IF-блоков по хинту и добыть 50 руды.
- M2: игрок может дополнить программу зарядкой по хинту и добыть 80 руды.
- M3: демо-программа запускается и добывает руду; оба дрона конфликтуют у `mine1`.
- M4: демо-программы работают, дроны останавливаются при разряде (ожидаемое поведение).
- В коде миссий нет устаревшего формата IF-условий — только `ConditionLeaf { left: FunctionCall }`.
- Все существующие unit-тесты проходят без изменений.

## Технические заметки

### Формат ConditionLeaf

Все IF-условия записываются в актуальном формате `types.ts`:

```ts
// пример: inventory > 0
{
  left:  { fn: 'Inventory', args: [{ kind: 'self' }] },
  operator: '>',
  right: { kind: 'number', value: 0 },
}

// пример: distance(base) = 0
{
  left:  { fn: 'Distance', args: [{ kind: 'self' }, { kind: 'entity', id: baseId }] },
  operator: '<=',
  right: { kind: 'number', value: 0 },
}
```

Составные условия (AND): `conditions: [leaf1, leaf2]`, `operators: ['AND']`.

### Distance и нулевое расстояние

Для сравнения с нулём использовать `<=` вместо `=` — надёжнее при возможной погрешности движения по тайлам.

### Порядок IF-блоков

Порядок определяет приоритет: зарядка → выгрузка → движение к базе → добыча → движение к шахте. Несколько условий могут сработать в одном тике, поэтому если они не конфликтуют по направлению — это нормально.

### commandSlots

Личные программы дронов в M1/M2 создаются с `commandSlots: 4`. Проверить при реализации: ограничивает ли это число top-level инструкций личной программы или только UI-слоты. `LOOP` с 4 IF внутри — 1 top-level инструкция.
