# Координаты клетки под курсором

**Статус:** planned
**Plan:** docs/superpowers/plans/2026-06-28-cell-coordinates-hud.md

## Зачем

Игрок программирует дронов и часто оперирует координатами клеток поля (например, `move({ x: 3, y: 10 })`). Сейчас узнать координату клетки можно только на глаз. Фича даёт быстрый способ увидеть и скопировать координаты любой клетки, чтобы вставлять их прямо в код дрона.

## Поведение

- При наведении курсора на пустую клетку поля её граница выделяется полупрозрачной заливкой + обводкой (цвет `#00d4ff`) поверх сетки.
- При наведении на клетку со зданием (шахта, база, зарядка) показывается **только обводка, без заливки** — чтобы не перекрывать спрайт здания.
- В правом нижнем углу появляется небольшой блок с координатами текущей клетки (`x: N  y: M`). Блок виден **только при ховере** над клеткой; когда курсор уходит с поля — блок прячется.
- Если курсор задерживается на одной клетке ≥ 1 секунды, рядом с блоком появляется попап с координатами и иконкой копирования.
- При клике на иконку в буфер обмена копируется строка `{ x: 3, y: 10 }` (валидный JS-объект). Кратковременный фидбэк «Скопировано».
- Всё работает **только внутри границ сетки**. Курсор над пустым фоном за сеткой — ни блока, ни рамки, ни попапа.
- При панорамировании камеры (зажата ЛКМ + движение) ховер, рамка и попап скрыты, чтобы не мешать.
- Координаты вычисляются с учётом зума и скролла камеры.

## Критерии готовности

- [ ] Наведение на пустую клетку показывает заливку + обводку; на клетку со зданием — только обводку.
- [ ] Блок координат появляется в правом нижнем углу при ховере.
- [ ] Координаты корректны при любом зуме/панораме (через `camera.getWorldPoint`).
- [ ] Блок и рамка прячутся за границами сетки и при зажатой кнопке мыши.
- [ ] Задержка ≥ 1 сек на клетке показывает попап с иконкой копирования.
- [ ] Копирование кладёт в буфер строку `{ x: N, y: M }`.
- [ ] Юнит-тест на `setHoveredCell` (дедупликация, null).
- [ ] E2E: ховер → блок с верными координатами; задержка → попап → копирование.

## Технические заметки

### Связь Phaser → React: Zustand + дедупликация по значению клетки

`pointermove` срабатывает десятки раз в секунду, но координата клетки меняется только при пересечении границы клетки (40px). Поэтому пишем в store **только при смене клетки** — не throttle по времени, а дедупликация по значению. Это даёт ре-рендер React ровно при смене клетки и согласуется с архитектурой проекта (`hoveredCell` — такое же состояние, как `selectedDroneId`, `stats`).

Интерактивные зоны Phaser (`add.zone().setInteractive()`) для каждой клетки — отклонено: 30×30 = 900 объектов, дорого и избыточно. Браузерный `IntersectionObserver` неприменим — клетки рисуются на одном `<canvas>`, отдельных DOM-узлов у них нет. Вместо этого — арифметика `floor(world / TILE_SIZE)` + сравнение с предыдущей клеткой.

### Phaser (`src/renderer/scenes/GameScene.ts`)

- Один переиспользуемый `Graphics` `hoverHighlight` (depth между тайлмапом и объектами): обводка `#00d4ff` всегда + полупрозрачная заливка только для пустых клеток.
- Тип клетки берём из `this._grid.getTile(cell.x, cell.y)` (уже доступен в `GameScene`). Значения `"mine" | "base" | "charger"` → здание → рисуем только обводку; `"empty"` → заливка + обводка. (Не нужен `getSlotRefAt`/World — достаточно типа клетки.)
- В обработчике `pointermove`:
  - если `pointer.isDown` → скрыть рамку, `setHoveredCell(null)`, выйти (это панорама);
  - `world = camera.getWorldPoint(pointer.x, pointer.y)`;
  - `cell = { x: floor(world.x / TILE_SIZE), y: floor(world.y / TILE_SIZE) }`;
  - если вне `grid` → `cell = null`;
  - если `cell` совпадает с `prevCell` → ничего не делать (дедуп);
  - иначе обновить `prevCell`, перерисовать/спрятать рамку, вызвать `setHoveredCell(cell)`.
- На `pointerout`/потерю фокуса канваса — сбросить в `null`.
- `TILE_SIZE` из `src/renderer/config.ts`. Утилиту `worldToGrid` не выносим — одна строка `floor` инлайн.

### Store (`src/shared/store/gameStore.ts`)

- Поле `hoveredCell: { x: number; y: number } | null` (default `null`).
- Action `setHoveredCell(cell)` с защитой от записи равного значения (сравнить x/y с текущим).

### React (`src/ui/overlays/CellCoordinatesHud.tsx` — новый)

- `useGameStore((s) => s.hoveredCell)`; если `null` — `return null`.
- Блок в правом нижнем углу: `position: absolute; bottom; right`, стиль по образцу `OreHud.tsx` / `MissionGoalButton.tsx` (monospace, палитра проекта).
- Таймер: на смену `hoveredCell` сбрасывать `setTimeout`; по 1 сек показать попап. Уход/смена клетки — сбросить таймер и попап.
- Копирование: `navigator.clipboard.writeText(`{ x: ${x}, y: ${y} }`)`, кратковременный фидбэк.
- Подключить в `src/App.tsx` рядом с прочими оверлеями.
