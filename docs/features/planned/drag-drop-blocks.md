# Drag & Drop блоков + визуальный рефактор редактора

**Статус:** planned

## Зачем

Редактор программ позволяет только добавлять/удалять инструкции — перемещать нельзя. Для сложных программ это создаёт неудобство: приходится удалять и заново добавлять блоки в нужном порядке. Также визуально редактор перегружен рамками и горизонтальными разделителями.

## Поведение

- Каждый блок имеет drag-handle `⠿`, появляющийся при наведении на строку
- Блоки перетаскиваются между любыми уровнями вложенности (кросс-контейнерный DnD)
- Контейнерные блоки (LOOP/IF/REPEAT) тянутся вместе с телом
- Во время drag: полупрозрачная копия следует за курсором, синяя линия показывает место сброса
- Блоки отображаются без рамок — только `borderLeft` показывает вложенность

## Критерии готовности

- [ ] Drag-handle `⠿` виден только при hover на строку блока
- [ ] Перетаскивание в рамках одного списка меняет порядок
- [ ] Перетаскивание между уровнями вложенности работает
- [ ] Контейнерный блок тянется целиком
- [ ] Блоки без рамок, вертикальная линия показывает вложенность
- [ ] Горизонтальные разделители в ConditionEditor убраны
- [ ] `npm test` проходит без ошибок
- [ ] `npm run type-check` без ошибок

## Технические заметки

- Библиотека: `@dnd-kit/core` + `@dnd-kit/sortable`
- Действие в store: `moveInstruction(programId, fromPath, toContainerPath, toIndex)`
- Каждый список инструкций (корень, body, then) получает свой `SortableContext`
- `DndContext` размещается в `ProgramEditor/index.tsx`
- Спек: `docs/superpowers/specs/2026-05-26-drag-drop-blocks-design.md`
