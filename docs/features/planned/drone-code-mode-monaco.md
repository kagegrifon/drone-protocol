# Code Mode (этап 2) — редактор Monaco и переключатель код/блоки

**Статус:** planned (после [drone-code-mode-core.md](drone-code-mode-core.md))

Спецификация: [2026-06-10-drone-code-mode-design.md](../../superpowers/specs/2026-06-10-drone-code-mode-design.md)

## Зачем

Дать игроку полноценный опыт написания кода: подсветка, автодополнение строго по API дронов,
проверка типов до запуска — всё внутри игры. Плюс переключатель «код / блоки» в настройках.
Садится на проверенное ядро этапа 1.

## Поведение

- В UI появляется редактор кода на базе **Monaco** (движок VS Code).
- Автодополнение и типы берутся из `drone-api.d.ts` (того же, что использует рантайм воркера):
  игрок видит `drone.moveTo()`, `drone.energy` и т.д. с JSDoc-подсказками.
- Синтаксические/типовые ошибки подсвечиваются до запуска миссии.
- В меню настроек — тумблер **«код / блоки»**, выставляет `behavior.source`.
- **Вне миссии** тумблер активен; **во время миссии** — заблокирован с пояснением.
- `source` фиксируется при старте миссии и не перечитывается до её конца.

## Критерии готовности

- [ ] Monaco встроен как React-компонент, код игрока сохраняется в `behavior.code`.
- [ ] `drone-api.d.ts` подключён в Monaco → автодополнение/типы по API дронов.
- [ ] Ошибки типов/синтаксиса видны в редакторе до запуска.
- [ ] Тумблер код/блоки в настройках: активен вне миссии, disabled + пояснение в миссии.
- [ ] E2E (Playwright): ввести код → запустить миссию → дрон выполняет цикл добычи; тест
      self-contained.
- [ ] `npm run type-check`, `npm test`, `npm run test:e2e` зелёные.

## Технические заметки

**Один источник правды для API.** `src/game/code/drone-api.d.ts` уже создан на этапе 1 для
рантайма. Здесь скармливаем его Monaco через `monaco.languages.typescript.*` — контракт не
дублируется.

**Фиксация `source` на старте миссии** — поведение уже заложено ядром этапа 1; здесь только
UI-блокировка тумблера во время миссии (читаем флаг «миссия активна» из стора).

**Новые файлы:** `src/ui/editor/CodeEditor/*`, изменения в экране настроек.
Monaco тянет вес в бандл — учесть при сборке (`npm run build`, base-path для деплоя уже
настроен).

**НЕ входит:** полноценный двойной UI-режим «блоки и код одновременно на экране» (пока
переключение через настройки), импорты/модули, шаринг кода.
