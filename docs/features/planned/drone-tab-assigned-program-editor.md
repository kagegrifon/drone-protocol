# Вкладка DRONE: редактор кода назначенной программы

**Статус:** planned

> Этап 1 из 2. Этап 2 — [code-debugging-line-highlight.md](code-debugging-line-highlight.md)
> (подсветка текущей строки) — строится поверх этого этапа.

## Зачем

На вкладке DRONE для **прикреплённой** (назначенной из library) программы сейчас
показывается только её имя и кнопка «открыть в библиотеке» — сам код увидеть и
редактировать на вкладке DRONE нельзя. Персональная же программа показывается в
полноценном `CodeEditor`.

Это асимметрия: пользователь не видит код активной программы дрона, когда она
прикреплена. Цель этапа — показывать код назначенной программы прямо на вкладке
DRONE в редакторе и дать его редактировать, симметрично персональной программе.

Этот этап самодостаточен (только UI, без изменений worker/simulation) и является
фундаментом для этапа 2: подсветка строки имеет смысл только когда код активной
программы виден на вкладке DRONE.

## Поведение

- На вкладке DRONE, когда у дрона есть назначенная программа, под её именем
  показывается `CodeEditor` с кодом этой программы.
- Код **редактируемый**: правки идут через тот же `setProgramCodeSource(programId, code)`.
  Так как программа общая, правки затрагивают все дроны с этой программой —
  это ожидаемое поведение (как при редактировании в LIBRARY/PROGRAM).
- Блок назначенной программы остаётся сворачиваемым (как персональный), чтобы
  не занимать место — переиспользовать существующий паттерн `personalExpanded`.
- Существующие кнопки (↗ «открыть в библиотеке», переключение radio на персональную)
  сохраняются.
- Когда назначенной программы нет — поведение прежнее (редактор персональной программы).

## Критерии готовности

- На вкладке DRONE у дрона с назначенной программой виден её код в редакторе.
- Редактирование кода назначенной программы сохраняется и применяется (дрон
  перезапускает сессию — поведение `setProgramCodeSource` не меняется).
- Существующее поведение персональной программы и кнопок не сломано.
- `npm run type-check` зелёный; e2e по редактору программ (если есть) проходят.

  редактором активной (назначенной, если есть; иначе персональной) программы.
