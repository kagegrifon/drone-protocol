# Свойства объектов мира в условиях IF

**Статус:** done

## Зачем

Сейчас условие `IF` оперирует фиксированным набором свойств самого дрона
(`ENERGY`, `INVENTORY`, `DEPOSIT`, `DISTANCE`), при этом правая часть сравнения
обязана быть числом. Это блокирует естественные сценарии:

- «инвентарь не максимум» (нужно `Inventory < InventoryMax`)
- «руда конкретной шахты кончилась» (нужно `Deposit(Mine#2)`)
- «я стою у этой шахты» (нужно `Distance(Self, Mine#1) = 0`)

Контекст усиливается фичей [atomic-actions](../done/atomic-actions.md):
после неё `MINE`, `DROP`, `CHARGE`, `MOVE_TO` стали пошаговыми, и
макроповедение игрок собирает через `LOOP` + `IF`. Богатые условия —
прямое продолжение этой линии.

Фича вводит универсальный конструктор условий через **встроенные функции**,
которые читают свойства объектов мира. Гибкость растёт, но мини-языка
программирования не появляется: leaf по-прежнему сводится к «функция оператор
операнд».

## Поведение

### Модель leaf'а

Каждый leaf условия IF имеет форму:

```
<функция>(<аргументы-объекты>)  <оператор>  <число | функция>
```

Логические комбинаторы `AND` / `OR` между leaf'ами и сам блок `IF` не
меняются.

### Доступные функции (MVP)

| Функция | Арность | Возвращает | Что значит |
|---|---|---|---|
| `Energy(объект)` | 1 | число | текущая энергия объекта |
| `EnergyMax(объект)` | 1 | число | максимум энергии объекта |
| `Inventory(объект)` | 1 | число | текущая загрузка ore |
| `InventoryMax(объект)` | 1 | число | вместимость инвентаря |
| `Deposit(шахта)` | 1 | число | оставшаяся руда в шахте |
| `Distance(a, b)` | 2 | число | manhattan-расстояние между a и b |

`Energy` / `EnergyMax` применимы только к объектам с компонентом `Energy`
(дроны). `Inventory` / `InventoryMax` — к объектам с компонентом `Inventory`
(дроны). `Deposit` — только к объектам с компонентом `Deposit` (шахты).
`Distance` — к любой паре объектов с компонентом `Position`.

Если функция применена к объекту, у которого нужного компонента нет,
интерпретатор возвращает `null`, и leaf даёт `false` (как сейчас при
отсутствии компонента).

### Объекты-аргументы

В аргументе функции выбирается один из:
- **`Self`** — символическая ссылка на дрона, исполняющего программу.
  Резолвится в runtime в `droneId` шага интерпретатора. Программа,
  ссылающаяся на `Self`, переносима между дронами.
- **Конкретная сущность миссии** из `EntityMeta[]` (например, `Mine #1`,
  `Base`, `Charger #1`).

### UX редактора leaf'а

Строка собирается слева направо:

```
[Функция ▾]  [Аргумент ▾]  ([Аргумент ▾])  [Op ▾]  [число | функция ▾]  [значение]  [×]
```

- Дропдаун «Функция»: 6 пунктов с иконками
  (🛣 Distance, ⚡ Energy / 🔋 EnergyMax, 📦 Inventory / 📦+ InventoryMax,
  ⛏ Deposit — финальные иконки уточняются при имплементации).
  Лейблы английские; локализация — будущая фича.
- Дропдауны аргументов появляются по арности функции. Для `Distance` —
  две подписи «от / до», для остальных — одна без подписи. Опции:
  `Self` сверху, далее отфильтрованные `EntityMeta` (фильтр зависит от
  функции — например, `Deposit` показывает только Mine'ы).
- Оператор — без изменений: `<, <=, =, >=, >`.
- Переключатель типа RHS: `[число ▾]` (числовой input) или
  `[функция ▾]` (вложенный блок «функция + аргументы», без оператора).
- Препросмотр строится «как код»: `Distance(Self, Mine#1) < 3 AND Inventory(Self) < InventoryMax(Self)`.

Свёрнутый чип IF-блока в дереве программы рендерится в том же
code-style, без пробелов внутри вызова функции: `Distance(Self,Mine#1)<3`.

### Что НЕ входит в фичу

- Проценты (`%`) — убираются. Игрок сам выражает «полный инвентарь» как
  `Inventory(Self) = InventoryMax(Self)`. Сравнение с долей вместимости
  (например, «инвентарь больше 80%») недостижимо без арифметики и в
  scope не входит.
- Предикаты (`IsSamePlace`, `IsCharging` и т. п.) — пока не нужны,
  «в той же клетке» выражается как `Distance(a, b) = 0`. Могут быть
  добавлены отдельной фичей.
- Блок `WHILE` и `BREAK` — отдельная фича. В текущем AST выйти из `LOOP`
  по условию нельзя; в рамках этой фичи условные циклы выражаются как
  «делать что-то, только если условие верно» через `LOOP { IF cond then [действие] }`,
  что соответствует пошаговой семантике atomic-actions (нет состояния —
  действие просто пропускается). Полноценный «делать до тех пор пока» —
  отдельная фича.
- AST-выражения, локальные переменные, арифметика на RHS — не вводим.
- Глубина вложения функций в RHS — строго 1 (RHS-функция не может
  содержать ещё одну функцию).

## Технические заметки

### Слой данных

```ts
type ObjectRef =
  | { kind: 'self' }
  | { kind: 'entity'; id: EntityId };

type FunctionName =
  | 'Energy' | 'EnergyMax'
  | 'Inventory' | 'InventoryMax'
  | 'Deposit'
  | 'Distance';

type FunctionCall =
  | { fn: 'Energy';       args: [ObjectRef] }
  | { fn: 'EnergyMax';    args: [ObjectRef] }
  | { fn: 'Inventory';    args: [ObjectRef] }
  | { fn: 'InventoryMax'; args: [ObjectRef] }
  | { fn: 'Deposit';      args: [ObjectRef] }
  | { fn: 'Distance';     args: [ObjectRef, ObjectRef] };

type Operand =
  | { kind: 'number';   value: number }
  | { kind: 'function'; call: FunctionCall };

interface ConditionLeaf {
  left: FunctionCall;
  operator: ConditionOperator;   // <, <=, =, >=, >
  right: Operand;
}
```

### Реестр функций

```ts
interface FunctionSpec {
  name: FunctionName;
  label: string;          // 'Distance', 'Energy', ...
  icon: string;           // '🛣', '⚡', ...
  arity: 1 | 2;
  argLabels: string[];    // ['от', 'до'] | ['']
  argFilter?: (entityId: EntityId, world: World) => boolean;
  evaluate: (resolved: EntityId[], droneId: EntityId, world: World) => number | null;
}
```

`evaluate` всегда получает уже резолвленные `EntityId` (Self → droneId).
Это держит спеки чистыми и упрощает unit-тесты.

### Поток evaluateLeaf

```
1. resolve(leaf.left)   → number | null
2. resolve(leaf.right)  → number | null  (число вернёт само себя; функция — через реестр)
3. если любая сторона null → false
4. иначе применить оператор
```

Глубина рекурсии всегда ≤ 1 (охраняется типом `Operand`).

### Совместимость и миграция

Программы не персистятся между сессиями (`localStorage` используется
только для размера панели в `BottomPanel.tsx`). Runtime-миграция не
нужна. Единственный затронутый объект кода — `mission3.ts:54`, leaf
которого переписывается вручную.

### Стоимость на тик

Идентична текущей: каждый leaf — это 1-2 `world.getComponent` + арифметика.
Регрессов производительности не ожидается.

### Расширяемость

Новая функция добавляется одной записью в `FUNCTIONS` и опциональным
`argFilter`. UI подхватывает автоматически. Поддержка предикатов
(будущая фича) — расширение `FunctionSpec` полем `returnType: 'number' | 'bool'`
и отдельный рендер leaf'а для bool-функций.
