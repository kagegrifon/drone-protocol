# Step-режим отладки дронов (этап B)

**Статус:** done
**Spec:** [step-debug stack trace](../../superpowers/specs/2026-06-29-step-debug-stack-trace-design.md) (этап B)
**Plan:** [step-debug stage B](../../superpowers/plans/2026-06-29-step-debug-stage-b-plan.md)

## Зачем

После этапа A (stack-trace + подсветка строк в модулях, см. [multimodule-debugging](../done/multimodule-debugging.md))
у игрока есть Call Stack и подсветка позиции, но нет способа **приостановить симуляцию и прогнать код
дрона по шагам**. Часто игрок хочет просто «подвигать по шагам и посмотреть, как работает» — не
дебажить баги, а наблюдать исполнение. Этап B даёт пошаговый просмотр.

## Поведение

- В верхней панели — кнопка **`⏯ Step Mode`** рядом с Play/Pause. Включение режима ставит игру на
  паузу и заливает кнопку оранжевым; выключение (или Continue) возвращает обычный режим.
- В step-режиме таб **DRONE** получает оранжевую точку-индикатор, а над хлебными крошками появляется
  полоса **STEP** с кнопками:
  - **Step action** — промотать тики до следующего действия выбранного дрона (следующее `await self.*`).
  - **Step tick** — продвинуть симуляцию ровно на один игровой тик.
  - **Continue** — выйти из step-режима и возобновить симуляцию.
- На паузе хлебные крошки и подсветка строки (этап A) показывают актуальную позицию дрона; Step action
  двигает позицию на следующее действие.
- **Нейминг — «Step Mode», без debug/breakpoint-лексики** в UI: режим про просмотр, не про отладку багов.

## Критерии готовности

- `GameController.stepDroneAction(droneId)` промотает тики до смены действия дрона с лимитом
  `MAX_STEP_TICKS`; no-op на `won`/`failed`.
- Флаг `isStepMode` в gameStore; кнопка Step Mode ставит/снимает паузу через `GameController`.
- Step-панель и точка-индикатор видны только в step-режиме; Step action/Step tick/Continue работают.
- `npm test`, `npm run type-check`, `npm run test:e2e` зелёные (новые unit `stepDroneAction`,
  `gameStore.stepMode`; e2e `step-mode`).

## Технические заметки

> Детали дизайна и решения дизайн-сессии — в спеке; пошаговая реализация — в плане (по ссылкам выше).

- Семантика паузы переиспользует существующие `GameController.pause()/step()/start()` и
  `gameStatus === "paused"`.
- «Действие дрона сменилось» детектируется по смене пары `(program.state, program.currentLine)`
  выбранного дрона — это ловит начало следующего `await self.*`.
- Доступ к `GameController` из вложенного `ProgramEditor` — через React-контекст `GameControllerContext`
  (без проброса пропсов через всю иерархию).
- Этап C (statement-level step + breakpoints) — отдельная будущая фича, в этап B не входит.
