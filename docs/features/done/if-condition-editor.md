# Редактор условий IF-блока

**Статус:** done

## Зачем

Сейчас условие в IF-блоке захардкожено как `INVENTORY_FULL` и не редактируется в UI. Пользователь не может задать ни тип условия, ни его параметры. Фича делает IF-блок полноценным: пользователь сам выбирает, что и с чем сравнивать.

## Поведение

### IF-блок в списке программы

- Если условие не задано — блок подсвечен оранжевым, показывает «условие не задано» и кнопку «✏️ задать».
- Если условие задано — показывает цветные чипы: каждое условие отдельным бейджем, между ними — оператор **И** или **ИЛИ**.
- Кнопка **✏️** открывает попап-редактор прямо под блоком.

Примеры чипов на блоке:
- `⚡ < 30%` — энергия меньше 30%
- `📦 = 100%` — инвентарь полон
- `⛏️ = 0 ед.` — депозит пуст
- `📍 База-1 ≤ 3 кл.` — расстояние до объекта не больше 3 клеток

### Попап-редактор условий

Открывается под IF-блоком по кнопке ✏️. Содержит:

1. **Список строк условий** — каждая строка: `[свойство] [ед.?] [оператор] [значение] [✕]`
2. **Переключатель И / ИЛИ** между каждыми двумя строками
3. **Кнопка «+ добавить условие»** — добавляет новую строку снизу (аналогично добавлению инструкций)
4. **Строка превью** — живая читаемая запись: `👁 Энергия < 30% И Инвентарь = 100%`
5. **Кнопки «Сохранить» / «Отмена»**

### Свойства (левая часть условия)

| Свойство | Иконка | Единицы | Объект |
|---|---|---|---|
| Энергия | ⚡ | % или ед. | — |
| Загрузка рудой | 📦 | % или ед. | — |
| Депозит | ⛏️ | только ед. | — |
| Расстояние | 📍 | только ед. (кл.) | выбирается как в MOVE_TO |

Для **Энергии** и **Загрузки рудой** появляется дополнительный селект `%` / `ед.` сразу после выбора свойства. Для **Депозита** и **Расстояния** — не появляется (нет максимума).

### Операторы

`<`, `<=`, `=`, `>=`, `>`

### Ограничения первой версии

- Максимум одна группа условий — без вложенности
- Оператор **НЕ** не реализуется
- Связь между всеми условиями выбирается попарно (И или ИЛИ между каждыми соседними)

## Технические заметки

### Новый тип `Condition` в `src/game/programs/types.ts`

Текущие 5 типов (`INVENTORY_FULL`, `INVENTORY_EMPTY`, `ENERGY_LOW`, `ENERGY_FULL`, `DEPOSIT_EMPTY`) заменяются на универсальную схему:

```typescript
type ConditionProperty =
  | { kind: 'ENERGY';    unit: '%' | 'abs' }
  | { kind: 'INVENTORY'; unit: '%' | 'abs' }
  | { kind: 'DEPOSIT' }
  | { kind: 'DISTANCE';  targetEntityId: EntityId };

type ConditionOperator = '<' | '<=' | '=' | '>=' | '>';

type Condition = {
  property: ConditionProperty;
  operator: ConditionOperator;
  value: number;          // всегда число; % = 0..100, abs = целое, кл. = целое
};
```

### Новый тип `ConditionBlock` в `src/game/programs/types.ts`

```typescript
type ConditionBlock = {
  type: 'IF';
  conditions: Condition[];
  operators: ('AND' | 'OR')[];  // длина = conditions.length - 1
  then: Instruction[];
  else?: Instruction[];
};
```

### Изменения в интерпретаторе (`src/game/programs/interpreter.ts`)

- Функция `evaluateCondition()` переписывается под новую схему
- Нужен доступ к позиции всех сущностей для `DISTANCE`
- Backward compatibility со старыми программами не нужна (нет сохранённых данных)
