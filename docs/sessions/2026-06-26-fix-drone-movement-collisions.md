# 2026-06-26 — Fix: дёрганое движение и коллизии дронов

## Цель

Ветка `fix-mooving` содержала перенесённые вручную правки движения из экспериментальной ветки. Задачи сессии:

1. Описать логику перенесённых изменений для дальнейшей разработки.
2. Исправить баг: два дрона едут в одну клетку — визуально «сливаются», затем одного отбрасывает назад.
3. Починить падающие unit-тесты (`npm run test`).

## Что сделано

### 1. Документация логики (задача 1)

В `DECISIONS.md` добавлены две записи (раздел «Архитектура»):

- **Continuous-движение** — путь больше не зачищается после каждого шага (`MovementSystem` убрал `movement.path = []`); дрон проходит весь буфер `path` подряд через `path.shift()`. Сопутствующее: `extendPathTail` в `planMove.ts` (пересчёт хвоста пути без рывка при новом `moveTo` на ходу), обрезка пути до текущего шага для не-`moveTo` действий в `CodeBehaviorDriver`.
- **Резервирование клеток** — модель разрешения коллизий (см. ниже).

### 2. Баг коллизии (задача 2)

**Корень:** `occupied` (для A*) содержал только текущие позиции дронов, не цели. Два дрона планировали путь в одну клетку, оба копили `progress` (визуальное «слияние»), затем один доезжал, второго отбрасывало назад со сбросом `path`/`progress`.

**Решение — резервирование целевой клетки** (выбор пользователя: «заранее не пускать», профиль «мало дронов, просторно»):

- Добавлено поле `Movement.reserved: Position | null` — клетка, в которую дрон едет (progress 0→1).
- `MovementSystem` перестроен: при ВЫЕЗДЕ (`progress === 0`) дрон проверяет занятость `path[0]`. Занята = там стоит дрон ИЛИ её зарезервировал движущийся дрон. Если занята — дрон не трогается (стоит, `progress = 0`, путь сохранён). Если свободна — резервирует и едет.
- Встречный обмен местами (swap) запрещён.
- Колонна вплотную трогается по очереди (клетка освобождается по факту прибытия).

**Визуальный фикс тряски:** ждущий дрон (`reserved == null`, `progress == 0`) в `GameScene` рисуется без t-интерполяции к целевой клетке (`targetCell = null`), иначе доля кадра `t` дёргала бы его вперёд-назад.

Событие `drone:blocked` больше не эмитится (оставлено в типах).

### 3. Тесты (задача 3)

- Исправлены 2 устаревших теста под continuous-движение (раньше проверяли зачистку `path` после атомарного шага).
- Блок «фикс гонки» переписан в «разрешение коллизий»: новые сценарии — две цели в одну клетку, отсутствие отбрасывания при медленной скорости, резервирование, цепочка A→B→C по очереди, запрет swap, проход после освобождения.
