## Результаты

- **E2E зелёные** (7/7). Падения `drone-controls.spec.ts` (pause/reset) в
  исходном прогоне пользователя оказались флейки от параллельной нагрузки

  получил рабочий доступ к объектам мира.

### Фикс Monaco (`fix:`-коммит)

Подтверждён предсказанный риск: в редакторе кода дрона `self`/`World` давали
ошибки. Причины: глобальный `self: Window` из `lib.dom` перебивал
`declare const self: DroneApi`, а `export {}` делал `drone-api.d.ts` модулем —
`self`/`World` становились невидимыми; top-level `await` тоже подчёркивался.

Решение в `monacoSetup.ts`: `lib: ["es2022"]` (без `dom`), `module: ESNext` +
`target: ES2022` + `moduleDetection: Force`. Значения enum — **числами**
(`9`/`99`/`3`): реэкспорт enum из monaco-editor берёт их из внешней версии
typescript, не совпадающей с ts.worker, поэтому `typescript.ScriptTarget.ES2022`
не применялся. Строковые формы (`"es2022"`) убирают ошибки диагностики, **но
ломают автодополнение** — поэтому остановились на числах. В `drone-api.d.ts`
убран `export {}` (файл стал амбиентным extraLib, `self`/`World` глобальны).

Итог (ручная проверка пользователя): автодополнение `self.`/`World.` работает,
`await` чист, дрон движется и майнит. type-check чистый.
