# Индикация ошибки в коде дрона

Дата: 2026-06-28
Ветка: `feat/error-indication`

## Цель

Когда в коде дрона (Code Mode) происходит ошибка выполнения (runtime-исключение
или таймаут синхронного участка), показывать явную индикацию: в списке дронов,
у активной программы в редакторе и у самого дрона на игровом поле.

## Результаты

Бэкенд ошибок уже работал (`program.codeError` ловится воркером и течёт в store).
Добавлена визуальная индикация и исправлен баг со сбросом:

1. **Сброс при перезапуске** — `CodeBehaviorDriver.step()` очищает
   `program.codeError` при создании новой worker-сессии. Раньше старая ошибка
   висела до следующего `error`/`finished`.
2. **DroneList** — статус `ERR` + красный значок ⚠ (тултип с текстом ошибки),
   красная точка-индикатор. Приоритет (ошибка → пауза → статус) вынесен в хелперы
   `indicatorColor`/`indicatorLabel`.
3. **ProgramEditor** — значок ⚠ у имени активной программы (виден и в свёрнутом
   блоке) + красная рамка блока. Компонент `ErrorBadge`, флаги
   `assignedHasError`/`personalHasError`.
4. **DroneSprite** — метод `setErrorState`: пульсирующий красный tint тела
   (`COLORS.DRONE_ERROR`), идемпотентный, с очисткой в `destroy`. Вызов в
   `GameScene` по `program?.codeError`.

Маркер строки в Monaco решили не делать: точной строки ошибки нет (рунтайм знает
лишь последний пройденный await).
