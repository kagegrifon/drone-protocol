# 2026-06-20 — Fix e2e CI (code-mode fallback)

## Цель
Исправить падающий в GitHub CI e2e-тест code-mode.spec.ts. Тест падал с таймаутом при поиске `Press Start`, браузер показывал белую страницу.

## Корневая причина

**Проблема:** `vite preview` в CI не знает о `base: "/drone-protocol/"` из конфига сборки. Отдаёт ассеты по `/assets/...`, но `dist/index.html` ссылается на `/drone-protocol/assets/...` → 404 → пустой `<div id="root"></div>` → React не инициализируется → белая страница.

**Почему раньше не видели:** 
- Локально в headed-режиме использовался существующий preview-сервер (повторный запуск)
- В CI `reuseExistingServer: false` поднимал preview впервые, без флага base

## Что исправлено

### 1. playwright.config.ts
- Добавлены константы `isCI`, `PORT`, `BASE_PATH`
- В CI: `npm run preview -- --port 4173 --base /drone-protocol/`
- baseURL подстраивается: CI → `/drone-protocol/`, dev → `/`
- `reuseExistingServer: !isCI` — dev переиспользует, CI поднимает новый

### 2. e2e/*.spec.ts (все 3 файла)
- `page.goto("/")` → `page.goto("")` для использования baseURL из конфига

### 3. .github/workflows/deploy.yml
- Добавлен `npm run build` перед `npm run test:e2e`

### 4. e2e/code-mode.spec.ts
- Финальный timeout: 30s → 50s (в CI на Ubuntu дрон медленнее, нужен запас)
