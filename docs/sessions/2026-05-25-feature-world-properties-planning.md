# 2026-05-25 — feature: планирование «свойства объектов мира в IF»

## Цель

Спланировать новую фичу: дать игроку возможность обращаться к свойствам
объектов мира в условиях `IF`-блока программ. Стартовая постановка пользователя
включала примеры в духе `while (self.position != позиции объекта) move` и
`while (загруженность < макс) MINE`. Реализация — в отдельной сессии.

## Что сделано

- Изучены текущая модель условий (`ConditionLeaf`, `ConditionProperty`),
  интерпретатор, редактор и места использования (`mission3.ts`).
- Через брейнсторминг (`superpowers:brainstorming`) проведено 4 итерации
  уточнений:
  1. Scope: остановились на варианте A — расширение условий, без блока `WHILE`
     и без AST-выражений.
  2. Работа с позицией: вариант с спец-оператором отвергнут в пользу
     представления через функции.
  3. Миграция старого формата: вариант M3 (единая модель, computed-функции)
     с дальнейшим переходом на «функции вместо свойств».
  4. Финальная модель — **встроенные функции в реестре**: `Energy`,
     `EnergyMax`, `Inventory`, `InventoryMax`, `Deposit`, `Distance`.
     Только числовые. Без процентов. Без предикатов (на потом).
- Дизайн презентован по секциям: модель данных, UX leaf'а, миграция,
  архитектура и тестирование. Каждая секция — с подтверждением пользователя.
- Написана спецификация [docs/features/planned/world-properties.md](../features/planned/world-properties.md):
  модель типов, реестр `FunctionSpec`, UX редактора, критерии готовности,
  технические заметки. Учтён контекст недавно слитой фичи
  [atomic-actions](../features/done/atomic-actions.md).
- Обновлён [docs/features/index.md](../features/index.md): добавлена запись
  про `world-properties.md`.
- Self-review спеки: устранено противоречие про несуществующий `BREAK`,
  уточнён баланс `mission3`, добавлена связка с atomic-actions.

## Ключевые решения

- **Self как символ.** В типах `ObjectRef = { kind: 'self' } | { kind: 'entity'; id }`,
  резолв в runtime в `droneId` исполнителя. Программы остаются переносимыми
  между дронами (важно для библиотечных программ из
  [personal-programs](../features/done/personal-programs.md)).
- **Реестр функций.** Интерпретатор не знает имён функций — спрашивает
  `FUNCTIONS[fn].evaluate(...)`. Новая функция = одна запись в реестре.
- **Проценты убраны.** «Полный инвентарь» = `Inventory = InventoryMax`.
  Сравнение с долей не требуется в MVP.
- **Глубина вложения = 1.** RHS-функция не может содержать другую функцию
  (охраняется типом `Operand`). Если когда-то понадобится — отдельная фича.
- **Без runtime-миграции.** Программы не персистятся, единственный
  затронутый leaf — в `mission3.ts:54`.

## Что НЕ вошло в эту сессию

- Реализация фичи — пойдёт в отдельной сессии.
- Блок `WHILE` / `BREAK` для условных циклов — отдельная фича.
- Предикаты вроде `IsSamePlace(a, b)` — отдельная фича.
- Локализация лейблов функций — отдельная фича.
