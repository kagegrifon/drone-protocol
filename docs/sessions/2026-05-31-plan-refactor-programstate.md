### 2. Разработан детальный план рефакторинга ✅

**Новый тип ProgramState:**
```ts
'idle' | 'running' | 'move' | 'mine' | 'drop' | 'charge'
```

**Ключевые упрощения:**
- Двойное присваивание → одиночное: `program.state = 'move'` вместо `state = 'waiting'; waitingFor = 'move'`
- Проверка состояния → прямая: `state === 'move'` вместо `state === 'waiting' && waitingFor === 'move'`
- Сброс состояния → одна строка: `state = 'running'` вместо `state = 'running'; waitingFor = undefined`
- UI метки → генерация без маппинга: `state.toUpperCase()` для (MOVE, MINE, DROP, CHARGE)
