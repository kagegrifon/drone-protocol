## Цель

Устранить дёрганое движение дрона (паузы/рывки между шагами) при сохранении возможности менять поведение между клетками.

## Проблема

Коммит `6078efc` попытался убрать паузу, заставив `MovementSystem` идти весь путь сам. Но это сломало модель: воркер перестал опрашиваться по пути, дрон идёт слепо — игрок не может изменить поведение между клетками (поехать заряжаться, сменить шахту). Плюс визуальный рывок назад.

Исходно (до `6078efc`) была одношаговая модель: после каждого шага программа возобновляется, может пересчитать следующее действие. Но между шагом и приходом нового `intent` от воркера (~15ms async) дрон стоит 1 тик (~100ms) — видимая дёрганость.

## Решение: Continuous Movement (path-following + replanning)

**Ментальная модель:** дрон движется непрерывно по пути, программа на каждом шаге подтверждает курс. Подтверждение приходит вовремя → движение без остановки. Не приходит / курс сменился / действие не move → плавная остановка на клетке.

**Механика:**
- `planNextStep` — планирует РОВНО следующий 1 шаг A* к цели (look-ahead). НЕ сбрасывает `progress`.
- `MovementSystem` шагает по `path[0]`; если после shift `path` не пуст → продолжает движение, `state` остаётся `move`; если пуст → `state=running`, управление программе (безопасная остановка).
- `CodeBehaviorDriver` при `intent moveTo(той же цели)` вызывает `planNextStep` (дописывает следующий шаг); при другой цели → `planMoveToPoint` (новый путь).
- Ранний resume на `drone:moved` шлётся сразу (~15ms) — воркер успевает вернуть новый `intent` до конца текущего шага, путь продлевается до физической остановки.

**Критерий «та же траектория»:** по целевой точке (`targetX/targetY`).

**Плюсы:** плавное движение (как 6078efc), но сохранена точка пересчёта поведения между клетками (как исходная одношаговая); индустриальный стандарт (RTS, Factorio).
