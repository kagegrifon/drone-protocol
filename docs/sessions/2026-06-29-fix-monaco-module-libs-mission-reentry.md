# 2026-06-29 — fix: Monaco module-libs переживают перезаход в миссию

## Цель

Починить баг с импортируемыми функциями: при перезаходе в миссию в Monaco
оставались типы прошлой миссии — автокомплит предлагал модули предыдущей
миссии, а импорты текущей подчёркивались красным как «Cannot find module».
При этом сам код исполнялся корректно (линкер на рантайме резолвит импорты из
актуального `registry`, а не из Monaco).

## Первопричина

Module-libs для Monaco хранятся в модульных синглтонах `lastModuleDts` и
`moduleLibsDisposable` в [`monacoSetup.ts`](../../src/ui/editor/CodeEditor/monacoSetup.ts).
Они живут весь срок жизни вкладки браузера и не сбрасывались при перезаходе в
миссию. `updateModuleLibs` имеет кэш-проверку `if (dts === lastModuleDts) return`:
при совпадении сгенерированного `.d.ts` двух миссий (типичный случай —
одинаковые по сигнатуре стартовые/одноимённые программы) проверка давала ложный
no-op, и `extraLib` прошлой миссии оставался зарегистрированным.

`GameController` при загрузке миссии пересоздаётся и переинициализирует store
свежим `registry`, но глобальное состояние Monaco-синглтона никто не трогал.

## Что сделано

- **Fix**: добавлена `resetModuleLibs()` в `monacoSetup.ts` — dispose текущего
  `extraLib` + сброс кэша `lastModuleDts`.
- **Подключение**: вызов `resetModuleLibs()` в `App.tsx` в loading-`useEffect`
  перед созданием `GameController` (точка загрузки любой миссии). Размещено в
  UI-слое, где импорт Monaco уместен, без нарушения границы game-слоя
  (`GameController` не должен импортировать UI-редактор).
- **Unit-тест** [`monacoSetup.test.ts`](../../src/ui/editor/CodeEditor/monacoSetup.test.ts):
  регистрация extraLib, кэш-no-op и регресс перезахода (после `resetModuleLibs`
  тот же набор модулей регистрируется заново). Мокается `monaco-editor` через
  `vi.hoisted`.
- **testid-ы** на кнопки `open-settings` (SimControls) и `back-to-missions`
  (AudioSettingsModal) — для будущих e2e на перезаход.

## Результаты

- `npm run type-check` — чисто.
- `npx vitest run` — 285/285 зелёные (новый тест на регресс инвалидации
  детерминированно красный без фикса, зелёный с фиксом).

## Не сделано / открытые вопросы

- E2E на перезаход не добавлен: попытки воспроизвести симптом через DOM-squiggly
  и через снятие `monaco.editor.getModelMarkers()` в e2e не дали красный тест
  без фикса. Прямая диагностика показала, что при перезаходе в **ту же** миссию
  `useEffect` в `ProgramEditor` (зависимость `[programs, registry]`) сам по себе
  переригистрирует module-libs со стартовым (пустым) набором и очищает старое.
  Симптом, по-видимому, проявляется на конкретном тайминге/переходе между
  **разными** миссиями. Регрессия инвалидации покрыта unit-тестом; сквозной
  симптом пользователь верифицирует вручную.
