# Drone Loop: Code Mode — программирование дронов на JS/TS

**Дата:** 2026-06-10
**Статус:** спецификация утверждена

## Контекст и зачем

Игра построена на «Accessible Programming Fantasy»: игрок собирает поведение дрона из
визуальных блоков (AST `Instruction[]`), которые пошагово исполняет интерпретатор
(`stepProgram` в `src/game/programs/interpreter.ts`). Вводим возможность писать **настоящий
JS/TS-код** — без самодельных обёрток над тем, что уже есть в языке.

Основная аудитория после изменения — **ученики-фронтендеры** (учатся на JS/TS), сам автор и
«все остальные, кому понравится». Настоящий код полезнее блоков именно для обучения
фронтендеров: даёт неограниченную выразительность (переменные, функции, любая логика) и
мгновенную визуальную обратную связь от симуляции.

**Блоки не удаляются** — это второй источник поведения «рядом, позже». Архитектура
проектируется так, чтобы код и блоки сходились к одной модели исполнения.

## Принятые решения

| Решение | Выбор | Почему / отвергнутая альтернатива |
|---|---|---|
| Аудитория | Фронтендеры + автор + «все, кому зайдёт» | Код как обучающий инструмент, не для хардкорщиков |
| Где пишется код | Встроенный **Monaco** | Игра самодостаточна, «открыл и играешь», автодополнение по API. Внешняя IDE отвергнута: тяжёлый онбординг, нет доступа к ФС из браузера, ломает самодостаточность |
| Стиль API | **async/await** (`await drone.moveTo()`) | Код, который реально пишут фронтендеры. tick-функция (Screeps) отвергнута как менее идиоматичная |
| Судьба блоков | Остаются, «рядом позже» | MVP — только код, но AST-режим не выпиливается |
| Тип поведения | Дискриминированное объединение по `source` | Однозначность: компилятор гарантирует валидную комбинацию полей |
| Переключатель код/блоки | В настройках, **только вне миссии** | У дрона живое состояние исполнения; менять источник на лету недетерминированно |
| Порядок работ | Сначала ядро, **потом Monaco** | Доказать детерминизм исполнения раньше UI |

## Архитектурный принцип — единая модель намерений

Текущий поток: `AST (Instruction[]) → stepProgram → выставляет program.state → системы исполняют`.

Ключевое наблюдение из `interpreter.ts`: `stepProgram` уже работает как **модель намерений** —
действие (`MOVE_TO`/`MINE`/...) выставляет `program.state` и делает `return`; системы
(Movement/Mining) доводят его за несколько тиков; control-flow делает `continue`. JS-режим
встаёт сюда естественно.

Вводим общий контракт **behavior driver** — источник поведения, доводящий дрона за тик до
«занят действием» (`state='move'/'mine'/...`) либо «свободен» (`idle`). Две реализации:

- **AstBehaviorDriver** — нынешний `stepProgram`, обёрнут, логика не меняется.
- **CodeBehaviorDriver** — новый, исполняет JS-код игрока через Web Worker.

`ProgramExecutionSystem` каждый тик выбирает driver по `source` дрона. **Системы
Movement/Mining/Energy, `state`-механизм, астар, Phaser — не меняются.**

## Исполнение JS-кода (CodeBehaviorDriver + Web Worker)

Код игрока исполняется в **Web Worker** → изоляция, таймаут против вечных циклов, async-мост.

Async/await + намерения:
```js
while (true) {
  await drone.moveTo(ore);
  await drone.mine();
  await drone.moveTo(base);
  await drone.drop();
}
```

Механика одного `await drone.moveTo(ore)`:
1. Воркер шлёт намерение `{ drone, action:'move', target:ore }` и зависает на промисе.
2. `CodeBehaviorDriver` в тике: принимает намерение → `program.state='move'` → `return`.
   Дальше как сейчас — `MovementSystem` ведёт дрона несколько тиков.
3. `state` вернулся в `idle` → driver шлёт воркеру «done» → промис резолвится → код едет до
   следующего `await`.

Между `await`-ами код игрока не «живёт» в игровом времени: мгновенно добегает до следующего
намерения и снова засыпает. Один `await` ≈ одно атомарное намерение ≈ один `return` в
`stepProgram`. **Детерминизм пошаговости сохраняется.**

Защита и границы:
- Таймаут на синхронный участок между `await`-ами (ловит `while(true){}` без await).
- API дрона — единственный канал; `Math.random`/таймеры/сеть недоступны/замоканы.
- Геттеры (`drone.energy`, `drone.inventory`) синхронны — снапшот состояния на старте тика.

## API дронов

**Действия (async, занимают тики):**
```ts
await drone.moveTo(target)   // дойти до сущности (астар как сейчас)
await drone.mine()           // добыть (MiningSystem)
await drone.drop()           // выгрузить
await drone.charge()         // зарядиться
await drone.wait(seconds)    // пауза
```

**Сенсоры (sync, снапшот тика)** — переиспользуют текущие функции условий
(`src/game/programs/functions.ts`):
```ts
drone.energy   drone.energyMax
drone.inventory   drone.inventoryMax   drone.freeSlots
distance(a, b)   deposit(target)
```

Всё, что выражали блоки IF/WHILE, выражается обычным JS (`if (drone.energy < 20)`).
`src/game/code/drone-api.d.ts` — единый контракт API: рантайм воркера + (позже) подсказки
Monaco. Один источник правды.

## Модель данных — однозначный тип

```ts
type DroneBehavior =
  | { source: 'block'; instructions: Instruction[] }   // как сейчас
  | { source: 'code';  code: string }
```
При `source:'code'` поля `instructions` нет в типе (и наоборот). Затрагивает
`src/game/programs/types.ts`, реестр программ, `src/shared/store/gameStore.ts`.

## Настройки и переключение

- Тумблер «код / блоки» в меню настроек выставляет желаемый `source`.
- **Во время активной миссии тумблер заблокирован** (disabled + пояснение): у дрона живое
  состояние исполнения (AST: `callStack`; код: припаркованная корутина в воркере).
- `source` фиксируется при старте миссии, не перечитывается до конца.

## Этапы реализации

Разбито на два плана (см. `docs/features/planned/`):

1. **drone-code-mode-core.md** — ядро без Monaco. Дискриминированный тип, `CodeBehaviorDriver`,
   воркер, async-API, выбор driver, таймаут. Код вводится временно (textarea/фикстура). Цель —
   доказать детерминизм.
2. **drone-code-mode-monaco.md** — Monaco-редактор, `.d.ts`-подсказки, тумблер код/блоки в
   настройках, блокировка в миссии, E2E.

## Verification

- **Unit (Vitest, без Phaser):** `await drone.moveTo` → `state='move'` + выход; «done»
  резолвит промис; таймаут ловит `while(true){}`; детерминизм — одинаковый код даёт одинаковую
  трассу `state`.
- **Эквивалентность:** одна задача блоками и кодом → идентичная трасса `state` по тикам.
- **E2E (Playwright, после Monaco):** ввести код → запустить миссию → дрон выполняет цикл
  добычи. Self-contained (`fullyParallel: true`).
- Команды: `npm test`, `npm run type-check`, `npm run test:e2e`, `npm run dev`.

## Что НЕ трогаем

`MovementSystem`, `MiningSystem`, `EnergySystem`, `state`-механизм, астар, Phaser-рендер,
логику `stepProgram` (только оборачиваем).
